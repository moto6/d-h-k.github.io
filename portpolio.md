# 모션디바이스 S/W 엔지니어
- 🗓 근무기간 : 2016.12 ~ 2019.01 (2년)
- 📨 주요 업무 : Middleware 개발, Firmware 개발

#### 1) Middleware 개발 : 자동차 시뮬레이터의 모터제어와 컨텐츠를 연동하는 Middleware 개발
- OOP Framework 도입 
  ```
  기존의 소스코드가 SuperLoop 속에 모든 Logic이 들어있어 기능추가할 때 마다 영향이 가는 side-effect가 심했고, 버그가 계속해서 한가지 변경을 위해서 소스코드 여기저기를 돌아다니면서 반복적인 작업을 해야하는 유지보수가 곤란한 상황이 있었습니다. 모든 장비별로 각각의 프로젝트와 빌드파일로 관리해야한 어려움도 있었습니다. 이를 해결하기 위해 OOP FrameWork를 도입했고, 그 결과 유지보수가 쉬워졌을 뿐 아니라 서로 다른 하드웨어와 시뮬레이터 장비라도 한가지 빌드파일에서 동작하는 통합미들웨어를 개발해 성공적으로 적용했습니다.
  ```
- 장비 제어 알고리즘 개발 
  ```
  컨텐츠에서 나오는 움직임만으로는 장비의 물리적인 가동범위를 온전히 활용하지 못하는 문제점이 있었습니다. 사실적인 움직임보다는 탑승객들의 재미를 극대화하기 위해 모터의 PID 게인을 튜닝하고 모터 이동명령에 비선형 필터를 추가해서 불쾌한 잔 진동을 줄이고 가속& 감속시 확실하고 큰 모션 피드백을 구현했습니다.
  ```
- 하드웨어 의존성 추상화 라이브러리 개발
  ```
  근무 중 기존에 사용하던 PCI-E 통신으로 연결되는 NativeAPI 모션 컨트롤러가 단종되었고, 이를 대체하기 위해서 Ethernet 케이블로 연결되어 DLL을 호출하는 방식의 컨트롤러를 도입해 시뮬레이터에 통합하는 작업을 진행했습니다. 통합하면서 기존과 다른 시퀀스로 동작하는 함수들을 추상화해서 기존에 동작하는 NativeAPI 에 1:1 대체할수 있도록 라이브러리를 작성했고, 제어하는 모터 변경시 Delay가 있었는데 모터 제조사 엔지니어와 협업을 통해 Delay 없이 제어할 수 있도록 DLL 을 수정해 받는 작업을 진행했습니다. 그 결과 OOP 미들웨어에 성공적으로 통합할 수 있었습니다.
  ```

#### 2) Firmware 개발 : embedded 분야 OS-less 환경 / MCU Firmware 개발
- 아케이드 게임 별 전용 조이스틱 firmware개발
  ```
  우주비행선, 눈썰매 등 제품별로 다르게 제작된 조이스틱의 스위치와 각도를 입력받아 컴퓨터에 USB JoyStick 신호로 보내는 장비를 개발했습니다. 기계팀, 게임팀, 기획팀과의 의사소통이 중요한 업무였고, 이를 통해서 비지니스 요구사항을 달성하는데 협업과 의사소통의 중요성을 배울 수 있었습니다.
  ```
- Gaming Wheel 용 firmware 개발
  ```
  자동차 레이싱 시뮬레이터가 회사의 주력 상품이였는데 상용 Force-Feedback Racing wheel 의 원가가 높고(100만원) 자주 고장나며, 해외제작사의 AS가 아쉬운 문제가 있어서 내부 개발을 진행하였습니다. 시뮬레이터와 동일한 750W AC 서보 모터를 사용했는데, 시뮬레이터와는 다르게 레이싱휠을 사용할 때 Force-FeedBack에 대한 반응속도 문제를 해결하지 못해 중단된 프로젝트입니다.
  ```
- 서비스 로봇 개발 프로젝트 
  ```
  시뮬레이터(게임기) 탑승을 유도하는 로봇 개발 프로젝트 (정부지원사업)
  음성, 영상, 제어신호 송·수신 서버 &클라이언트용 S/W 개발, Motor Controller Firmware 개발
  ```
